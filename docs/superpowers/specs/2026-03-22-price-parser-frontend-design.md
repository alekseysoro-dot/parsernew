# Price Parser — Frontend Design Spec

**Дата:** 2026-03-22
**Этап:** Шаг 1 из 6 (Фронтенд)
**Файл реализации:** `reverens/price-parser.html` (один HTML-файл)

---

## Контекст

Учебный проект. Цель — мониторинг цен на товары на маркетплейсах (Wildberries, Ozon, Яндекс Маркет, DNS, МВидео). Реализуется поэтапно с использованием ИИ-агентов. Текущий этап — только фронтенд: интерактивный UI без бэкенда.

Референс: существующий прототип `reverens/price-parser.html`.

## Архитектура: три состояния одного HTML-файла

```
index.html (один файл)
├── Состояние 1: Список товаров        (drawer закрыт)
├── Состояние 2: Детали товара          (drawer открыт справа)
└── Состояние 3: Настройки             (модальное окно поверх)
```

Переходы управляются через JS: `classList.toggle`, `style.display`. Никакого роутинга, никаких фреймворков.

## Состояние 1 — Главный экран (список товаров)

### Layout
```
┌─────────────────────────────────────────────────┐
│  HEADER: Logo | Status badge | ⚙ | + Добавить  │
├────────────┬────────────────────────────────────┤
│  SIDEBAR   │  MAIN CONTENT                      │
│  300px     │  flex                              │
│            │  Search | ↻ | ▶ Парсить           │
│  Магазины  │  Stats ×4                          │
│  Все       │  [Parsing bar — скрыт по умолч.]  │
│  WB        │  Table                             │
│  Ozon      │                                    │
│  ЯМ        │                                    │
│  DNS       │                                    │
│  MVideo    │                                    │
│            │                                    │
│  Фильтры   │                                    │
│  Снизились │                                    │
│  Выросли   │                                    │
└────────────┴────────────────────────────────────┘
```

### Шапка (Header)
- Логотип + название "Price Parser"
- Статус-бейдж: пульсирующая точка + "Парсер активен"
- Иконка ⚙ — открывает модальное окно настроек
- Кнопка "＋ Добавить товар" — открывает модалку добавления (из прототипа)

### Сайдбар
- Секция "Магазины": список с иконками, активный — фиолетовый фон
- Кнопка "+ Добавить магазин" — dashed border, заглушка
- Секция "Быстрые фильтры": Снизились (зелёный), Выросли (красный)

### Статистика (4 карточки)
| Карточка | Описание |
|---|---|
| Товаров | Общее число. Число — monospace, крупно |
| Снизились | Количество. Число — зелёный цвет |
| Выросли | Количество. Число — красный цвет |
| Обновлено | Время последнего парсинга. Monospace |

### Строка поиска + действия
- Поле поиска с иконкой ⌕ (фильтрует таблицу на лету)
- Кнопка "↻ Обновить" — secondary
- Кнопка "▶ Парсить" — primary accent, запускает анимацию прогресс-бара

### Прогресс-бар парсинга
- Скрыт по умолчанию (`display:none`)
- Показывается при нажатии "▶ Парсить"
- Спиннер + текст прогресса + progress bar с gradient fill
- Автоматически скрывается по завершении

### Таблица товаров

Колонки: **Товар** | **Цена** | **Изменение** | **Магазин** | **Проверено** | **Действия**

- Строки кликабельны (`cursor: pointer`) — клик открывает drawer
- Активная строка: `border-left: 2px solid var(--accent)` + `background: rgba(108,92,231,0.06)`
- Цена — monospace, жирный
- Бейдж изменения: `▼ −5.2%` зелёный / `▲ +2.1%` красный / `— 0%` серый
- Действия в строке: ↻ обновить, ✕ удалить

### Пустое состояние
- Когда товаров нет: иконка + текст "Нет товаров. Добавьте первый." + кнопка

## Состояние 2 — Drawer детали товара

Открывается при клике на строку таблицы. Выезжает справа, список сужается.

### Поведение
- Ширина drawer: `380px`, фиксированная
- Список сужается: `grid-template-columns: 1fr 380px`
- Анимация: `transform: translateX(100%)` → `translateX(0)`, 200ms ease-out
- Закрытие: кнопка ✕ в шапке drawer ИЛИ клик вне drawer (на список)

### Содержимое drawer (5 блоков сверху вниз)

**1. Шапка**
- Название товара (жирный, 15px)
- SKU + магазин (monospace, серый, мелко)
- Кнопка ✕ (правый верхний угол)

**2. Текущая цена + изменение**
- Крупная цена: monospace, 24px, жирный
- Дельта: `▼ −5 010 ₽` и `−5.2% за 7 дней` (зелёный/красный)

**3. График истории цен**
- Чистый SVG (без библиотек)
- Линейный график за 30 дней
- Gradient fill вниз (accent → transparent)
- Подпись оси X: −30 дн / −14 дн / −7 дн / Сейчас
- Статические тестовые данные на шаге 1

**4. Цены по магазинам**
- Список всех магазинов с ценой
- Лучшая цена: зелёный текст + фиолетовая рамка
- Остальные: серый текст

**5. Действия**
- "↗ Открыть" — ссылка на страницу товара в магазине (заглушка)
- "✎ Изменить" — открывает модалку редактирования
- "✕" — удалить товар (красная иконка)

## Состояние 3 — Модальное окно настроек

Открывается по клику ⚙ в шапке. Поверх любого текущего состояния.

### Поведение
- Overlay: `rgba(0,0,0,0.6)` + `backdrop-filter: blur(4px)`
- Закрытие: кнопка ✕, кнопка "Отмена", клик по overlay
- Данные сохраняются в `localStorage`

### Секции

**Парсинг**
- Интервал обновления: выбор из [30 мин | 1 ч | 6 ч | 24 ч] (pill-кнопки)
- Автозапуск по расписанию: toggle-переключатель

**Уведомления**
- Порог снижения цены: числовое поле + "%" (input, мин. 1, макс. 99)
- Push-уведомления: toggle (заглушка, подпись "подключается на шаге 3")

**Данные**
- Глубина истории: [7 дн | 30 дн | 90 дн] (pill-кнопки)
- Экспорт CSV: кнопка-заглушка

**Кнопки:** "Отмена" (secondary) | "Сохранить" (primary)

## Интерактивность (JavaScript)

```
Функции:
- openDrawer(productId)     — открыть drawer, подсветить строку
- closeDrawer()             — закрыть drawer, снять подсветку
- openSettings()            — показать модалку настроек
- closeSettings()           — скрыть модалку настроек
- openAddProduct()          — модалка добавления (из прототипа)
- startParsing()            — показать прогресс-бар, анимация
- filterTable(query)        — фильтрация строк таблицы по строке поиска
- selectStore(storeId)      — фильтрация по магазину
- saveSettings()            — сохранить в localStorage
- loadSettings()            — загрузить из localStorage при старте
- renderChart(data)         — отрисовать SVG-график в drawer
```
